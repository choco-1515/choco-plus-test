# Choco-Tube-Plus Projectについて

- Choco-Tubeではコードの汚さや利用のしづらさが目立ったためアップグレードしたChoco-Tube-Plusを作ることにしました。
- 後継機といえるかは正直微妙ですが()
- Choco-Tube-Plusは、広告なしでYouTubeの動画を楽しむための、プライベート・フレンドリーなビデオストリーミングアプリケーションという名義です。

## 現状の問題点

- invidiousによる動画取得はサムネイルオーバーレイ(動画の長さ)、視聴数、投稿時間の情報を返さないことが多い<br>*投稿時間は取得できても~ago形式<br>*invidiousを利用するのは日本以外のトレンド、invidious優先検索
- チャンネルページでの動画などをもっと読み込むのボタンが壊れている

## 現時点で考えていること
- プロキシサムネイルの種類が多いためどうにかきれいにまとめる
- ストリーム再生の中に3つの項目<br>Invidiousストリーム<br>APIによるストリーム取得<br>yt-dlp<br>HJS再生
- サムネイルボタンで多くの方法によるサムネイル取得
## 3月7日時点の進み具合　

### 検索機能
- **複数検索方式のサポート**
  - **Invidious優先**: オープンソースのYouTubeフロントエンド「Invidious」を使用（デフォルト）
  - **YouTube API優先**: 公式のYouTube Data APIを使用
  - ページネーション対応

### トレンド表示機能
- **複数地域のトレンド動画表示**
  - 日本（JP）- wakameリポジトリから取得
  - アメリカ（US）
  - イギリス（GB）
  - 韓国（KR）
  - ドイツ（DE）
  - フランス（FR）
- Invidiousインスタンスを利用した安定した取得

### サムネイル取得プロキシ機能
3つの方式から選択可能：
1. **img.youtube.com** - 公式YouTube画像ホスト
2. **i.ytimg.com** - YouTube画像ホスト（別パス）
3. **自サーバー（Self-hosted）** - サーバーを経由したプロキシ取得（デフォルト）

### ユーザー設定保存機能
ブラウザのクッキーを使用して以下の設定を自動保存：
- **サムネイル取得方法** - デフォルト：自サーバー
- **検索優先方式** - デフォルト：Invidious優先

### UI/UX機能
- レスポンシブデザイン対応
- グリッドレイアウト表示
- ダークテーマ
- ナビゲーションバー（ホーム・トレンド）
- ヘッダー検索バー

### 動画視聴機能
- 個別動画ページ
- YouTubeプレイヤー埋め込み対応

## 技術スタック
- **バックエンド**: Python Flask
- **フロントエンド**: HTML, CSS, Jinja2
- **API連携**:
  - YouTube Data API
  - Invidious API
  - wakame トレンド JSON
